Infrastructure Virtualisée

Mise en place d'une infrastructure réseau virtualisée complète avec serveurs, DNS, Active Directory et gestion centralisée

L'objectif de ce projet est de mettre en place une infrastructure réseau virtualisée complète. L'infrastructure est composée de :


  • Serveurs Proxmox en cluster pour la virtualisation
  • Configuration réseau avec routeur CISCO et switch HP
  • Serveur Active Directory pour la gestion des utilisateurs
  • Serveurs DNS autoritaires et redirecteurs
  • API Proxmox avec PowerShell pour l'automatisation

Pour développer cette infrastructure, nous avons eu une période de 9 semaines en binôme, réalisant ces différentes étapes :


  • Installation et configuration de Proxmox
  • Création de templates Windows 10 et Windows Server 2022
  • Mise en cluster des serveurs
  • Configuration du routeur CISCO et switch HP
  • Mise en place de la NAT et des VLANs
  • Déploiement des services DNS et Active Directory
  • Automatisation via l'API Proxmox
  • Monitoring graphique de l'infrastructure

L'infrastructure est organisée en plusieurs zones réseau distinctes avec des VLANs dédiés. Le cluster Proxmox permet une gestion centralisée des machines virtuelles avec migration automatique.
Les serveurs DNS assurent la résolution de noms pour la zone r304-23.sae.proj et la redirection vers les DNS externes. Le serveur Active Directory gère l'authentification sur la zone masae.local.

Configuration Réseau

L'infrastructure réseau s'appuie sur une configuration NAT avec plusieurs interfaces et VLANs.


Configuration du routeur CISCO :


Interface Fa0/0 : ip nat outside

Interfaces Fa0/1.10 à Fa0/1.40 : ip nat inside


Serveurs DNS :


ns1 : Serveur DNS autoritaire sur la zone r304-23.sae.proj

ns2 : Serveur DNS redirecteur vers 10.1.147.199

Bilan personnel

Ce projet m'a permis de découvrir concrètement la mise en place d'une infrastructure réseau complète, de l'installation des serveurs Proxmox jusqu'à l'automatisation via PowerShell, en passant par la configuration des services DNS et Active Directory. Malgré la complexité du projet et les difficultés rencontrées avec la NAT et l'interface graphique, le travail en binôme sur 9 semaines a été enrichissant pour l'apprentissage mutuel et la répartition des tâches. Cette expérience m'a donné une vision concrète des enjeux d'une infrastructure d'entreprise et renforcé mes compétences en administration système et virtualisation.
